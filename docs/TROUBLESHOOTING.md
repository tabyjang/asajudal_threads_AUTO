# 문제 해결 가이드

## "사용할 수 없는 기능" 에러

### 원인
Meta Developer 앱 설정이 완료되지 않았거나 앱이 제대로 초기화되지 않았을 때 발생합니다.

---

## 해결 방법 1: 앱 기본 설정 확인

### 1단계: 앱 대시보드로 이동
- https://developers.facebook.com/apps
- 만든 앱 클릭

### 2단계: 앱 설정 > 기본 설정
왼쪽 메뉴에서 "설정 > 기본 설정" 클릭

**필수 입력 사항 확인:**
- ✅ 앱 이름
- ✅ 연락처 이메일
- ✅ 앱 도메인: `localhost` 입력 (또는 비워두기)

### 3단계: 앱 모드 확인
- 앱이 "개발 모드"인지 확인
- 상단에 "개발" 또는 "Development" 표시 확인

### 4단계: 역할 추가
"역할" 메뉴에서:
- 본인이 "관리자" 또는 "개발자" 역할인지 확인
- 테스터 추가 가능 (선택사항)

---

## 해결 방법 2: 새 앱으로 다시 시작 (권장)

때로는 새로 만드는 게 더 빠릅니다:

### 1. 기존 앱 삭제 (선택)
- 앱 대시보드 > 설정 > 고급 > 앱 삭제

### 2. 새 앱 생성
- "앱 만들기" 클릭
- **앱 유형**: "비즈니스" 선택 ⭐ (중요!)
- 앱 이름: 간단하게 (예: "Test App")
- 연락처 이메일 입력

### 3. Threads 제품 추가 건너뛰기
처음에는 Threads 제품을 추가하지 마세요!

### 4. Graph API Explorer로 바로 이동
- https://developers.facebook.com/tools/explorer
- 새로 만든 앱 선택
- "Generate Access Token" 클릭
- 권한 선택:
  - `instagram_basic` (Threads는 Instagram API 사용)
  - `instagram_content_publish`

---

## 해결 방법 3: Instagram Graph API 사용 (대안)

Threads API가 안 되면 Instagram Graph API를 사용할 수도 있습니다:

### 필요한 것
1. Instagram 비즈니스 계정 또는 크리에이터 계정
2. Facebook 페이지 (Instagram과 연결)

### 장점
- 더 안정적
- 문서가 더 많음
- 설정이 상대적으로 쉬움

### 단점
- Threads가 아닌 Instagram에 게시됨

---

## 해결 방법 4: 시간 기다리기

Meta Developer는 때때로 앱 정보를 동기화하는 데 시간이 걸립니다:

1. 앱을 만든 후 **30분~1시간** 정도 기다리기
2. 브라우저 캐시 삭제
3. 시크릿 모드로 다시 시도

---

## 해결 방법 5: 더 간단한 대안 사용

### A. Zapier 사용
- Threads 연동 지원
- 코딩 불필요
- 유료 (월 $20~)

### B. IFTTT 사용
- 간단한 자동화
- 무료 플랜 있음
- Threads 지원 여부 확인 필요

### C. Buffer / Hootsuite
- 소셜 미디어 관리 도구
- Threads 스케줄링 지원
- 유료

## 빠른 체크리스트

현재 상황을 확인해보세요:

- [ ] Facebook 계정이 있나요?
- [ ] Threads 앱에 계정이 있나요?
- [ ] Meta Developer 계정을 만들었나요?
- [ ] 앱을 생성했나요?
- [ ] 앱이 "개발 모드"인가요?
- [ ] 본인이 앱의 관리자/개발자인가요?

하나라도 체크 안 되면 해당 단계부터 다시 시작하세요.

---

## 그래도 안 되면

다음 정보를 확인해주세요:

1. **어느 단계에서 막혔나요?**
   - 앱 생성
   - 토큰 발급
   - API 호출

2. **어떤 에러 메시지가 나오나요?**
   - 스크린샷 찍어서 확인

3. **사용 중인 계정:**
   - Facebook 계정 있음
   - Threads 계정 있음
   - Instagram 계정 있음

위 정보를 알려주시면 더 정확하게 도와드릴 수 있습니다!
