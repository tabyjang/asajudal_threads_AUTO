# Threads 자동화 실전 운영 가이드

## 일주일 워크플로우

### 일요일 저녁 (2-3시간) - 콘텐츠 준비의 날

#### 1단계: 다음 주 콘텐츠 기획 (30분)

**체크리스트**:
- [ ] 다음 주 특별한 날 확인 (공휴일, 기념일)
- [ ] 12띠별 주간 키워드 정하기
- [ ] 일일 3-5개 게시물 스케줄 짜기

**요일별 추천 콘텐츠**:
```
월요일: 주간 운세 미리보기 + 월요병 극복 팁
화요일: 띠별 운세 + 실전 조언 (이직/사업)
수요일: 사주 미니 지식 + 퀴즈
목요일: 궁합 이야기 + 실제 상담 사례
금요일: 주말 운세 미리보기 + 이벤트
토요일: 특집 콘텐츠 (연애운 최고 띠 등)
일요일: 다음 주 운세 + 휴식 콘텐츠
```

#### 2단계: 콘텐츠 작성 (1시간)

**엑셀/구글 시트 활용**:

| 날짜 | 시간 | 유형 | 제목 | 내용 | 이미지 | 해시태그 |
|------|------|------|------|------|--------|----------|
| 12/24 | 07:00 | 운세 | 오늘의 운세 | ... | img1.jpg | #운세 |
| 12/24 | 13:00 | 팁 | 갑목일주 | ... | img2.jpg | #사주 |

**시간 배치 전략**:
- **07:00** - 오늘의 운세 (출근길 타겟)
- **12:00-13:00** - 미니 팁 (점심시간)
- **18:00-19:00** - 퀴즈/참여형 (퇴근길)
- **21:00** - 심화 콘텐츠/사례 (저녁 여유시간)

#### 3단계: 이미지 제작 (1시간)

**Canva 템플릿 활용**:

1. **Canva 접속**: canva.com
2. **템플릿 만들기**:
   - 크기: 1080x1080 (정사각형)
   - 브랜드 컬러 설정 (예: 보라, 금색)
   - 폰트 고정 (2-3종류만)

3. **콘텐츠별 템플릿**:
   ```
   📊 일일 운세:
   - 배경: 그라디언트
   - 12띠 아이콘
   - 별점 표시

   💡 미니 팁:
   - 단색 배경
   - 큰 제목
   - 핵심 키워드 강조

   🎯 퀴즈:
   - 밝은 배경
   - 4지선다 표시
   - 물음표 아이콘
   ```

4. **일괄 다운로드**:
   - 파일명 규칙: `2024-12-24-morning.png`
   - 폴더 정리: `images/2024-12/`

#### 4단계: 이미지 호스팅 (15분)

**방법 1: Imgur (추천, 가장 쉬움)**

```bash
1. imgur.com 접속
2. "New post" 클릭
3. 이미지 드래그앤드롭 (한 번에 여러 개 가능)
4. 업로드 완료 후 각 이미지 우클릭
5. "Copy link (direct link)" 선택
6. 엑셀에 붙여넣기
```

**방법 2: GitHub (무료, 무제한)**

```bash
1. GitHub 저장소 생성: threads-images
2. images/ 폴더에 파일 업로드
3. 파일 클릭 → "Raw" 버튼
4. URL 복사
   예: https://raw.githubusercontent.com/tabyjang/threads-images/main/image1.png
```

**방법 3: 본인 사이트 (asajudal.com)**

```bash
1. FTP로 웹서버에 업로드
2. 직접 접근 URL 확인
   예: https://asajudal.com/images/fortune1.png
```

#### 5단계: CSV 파일 작성 (15분)

**엑셀에서 작성 후 CSV로 내보내기**:

```csv
date,time,content_type,text,image_url,hashtags
2025-12-24,07:00,fortune,"🌅 오늘의 띠별 운세 [전체 내용]",https://i.imgur.com/abc123.jpg,"#오늘의운세 #띠별운세"
2025-12-24,13:00,tip,"💡 사주 미니 지식 [내용]",https://i.imgur.com/def456.jpg,"#사주 #일주"
```

**주의사항**:
- 큰따옴표 안에 텍스트 넣기
- 줄바꿈은 실제 엔터가 아닌 `\n` 사용
- 이미지 없으면 빈칸으로 (쉼표 두 개: `,,`)

## 서버 운영

### 옵션 1: PC 상시 가동 (무료, 권장)

**설정**:
```bash
# 1. 스케줄러 실행
npm run scheduler

# 2. PC 절전 모드 해제
# Windows: 설정 > 시스템 > 전원 > "절전 모드" 해제

# 3. 백그라운드 실행 (선택)
# VS Code 터미널 최소화만 해도 OK
```

**장점**:
- 완전 무료
- 실시간 로그 확인 가능

**단점**:
- PC를 꺼두면 작동 안 함
- 전기세 약간 증가

### 옵션 2: 클라우드 서버 (유료, 안정적)

#### 2-1. AWS EC2 (프리티어)

**설정**:
```bash
# 1. AWS EC2 프리티어 가입
# 2. Ubuntu 서버 생성
# 3. 프로젝트 업로드
git clone https://github.com/tabyjang/asajudal_threads_AUTO.git
cd asajudal_threads_AUTO
npm install

# 4. .env 파일 설정
nano .env
# ACCESS_TOKEN, THREADS_USER_ID 입력

# 5. PM2로 상시 실행
npm install -g pm2
pm2 start auto-scheduler.js --name threads-scheduler
pm2 save
pm2 startup
```

**비용**:
- 프리티어: 무료 (12개월)
- 이후: 월 $5-10

#### 2-2. Heroku (쉬움)

```bash
# 1. Heroku CLI 설치
# 2. 프로젝트 배포
heroku login
heroku create asajudal-threads
git push heroku master

# 3. 환경 변수 설정
heroku config:set ACCESS_TOKEN=your_token
heroku config:set THREADS_USER_ID=your_id
```

**비용**: 월 $7

#### 2-3. 라즈베리파이 (일회성 비용)

**장점**:
- 한 번 사면 끝 (약 7만원)
- 전기세 거의 안 나옴
- 집에서 24시간 가동

## 콘텐츠 DB 구축

### 재사용 가능한 콘텐츠 라이브러리

**폴더 구조**:
```
content-library/
├── daily-fortune/
│   ├── template1.txt
│   ├── template2.txt
│   └── template3.txt
├── tips/
│   ├── 갑목일주.txt
│   ├── 을목일주.txt
│   └── ...
├── quiz/
│   ├── quiz1.txt
│   └── quiz2.txt
└── images/
    ├── fortune/
    ├── tips/
    └── quiz/
```

**템플릿 예시**:
```
파일: daily-fortune/template1.txt

🌅 [{날짜}] {요일} 오늘의 운세

🐭 쥐띠: {내용}
💼 업무운 {별점} 💰 재물운 {별점}

🐮 소띠: {내용}
💼 업무운 {별점} 💰 재물운 {별점}

...

👉 내 상세 사주 분석
프로필 링크 확인 ✨
```

**자동 생성 스크립트** (선택):
```python
# generate_fortune.py
import datetime
import random

def generate_daily_fortune(date):
    template = load_template('daily-fortune/template1.txt')

    fortunes = {
        '쥐': random.choice(fortune_texts['rat']),
        '소': random.choice(fortune_texts['ox']),
        # ...
    }

    return template.format(
        날짜=date.strftime('%m월 %d일'),
        요일=get_weekday(date),
        **fortunes
    )
```

## 트러블슈팅

### 문제 1: 게시물이 안 올라가요

**체크리스트**:
```bash
# 1. 스케줄러가 실행 중인가?
# 터미널 확인

# 2. CSV 시간이 올바른가?
# 현재 시각 기준 ±5분 이내

# 3. posted_log.json 확인
cat posted_log.json
# 이미 게시됐으면 다시 안 올라감

# 4. 로그 확인
# 터미널에 에러 메시지 확인
```

**해결**:
```bash
# 캐시 삭제
rm posted_log.json

# CSV 시간 수정
nano content_schedule.csv

# 스케줄러 재시작
npm run scheduler
```

### 문제 2: 이미지가 업로드 안 돼요

**원인**:
- 이미지 URL이 리다이렉트
- HTTPS가 아닌 HTTP
- 파일이 너무 큼 (>8MB)
- 이미지 형식 미지원 (GIF, WebP 등)

**해결**:
```bash
# 1. 이미지 URL 직접 브라우저에서 열어보기
# → 바로 이미지가 보여야 함

# 2. URL 체크
# ✅ https://i.imgur.com/abc123.jpg
# ❌ https://picsum.photos/800/800?random=1

# 3. 이미지 최적화
# - 사이즈: 1080x1080
# - 포맷: JPG 또는 PNG
# - 용량: <2MB 권장
```

### 문제 3: 토큰이 만료됐어요

**증상**:
```
Error: Invalid OAuth access token
```

**해결**:
```bash
# 1. Graph API Explorer에서 새 토큰 발급
# https://developers.facebook.com/tools/explorer

# 2. .env 파일 수정
nano .env
# ACCESS_TOKEN=새로운_토큰

# 3. 스케줄러 재시작
npm run scheduler
```
